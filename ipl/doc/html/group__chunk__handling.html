Detailed Description

Adds a callback function to provide single data chunks instead of the complete IPF data. Useful if there is not enough RAM to store the complete IPF file.

Macro Definition Documentation

#define IPL_DATACHUNK_SIZE   2048

Size of data chunks in bytes. 0 means that the complete IPF file is stored in the memory.

Smallest value (besides 0) is 2048. If the value is unequal to 0, the additional callback function Ipl_ProvideDataChunk() is used for the application to provide the respective data chunk. If the value is 0, the complete IPF data is referred in the Ipl_Prog() call (no callback is used).
